About this sample project

This sample project is to demonstrate a simple web framework, which is fully written in java. It has a three-tier architecture.

  • Presentation Layer: Bootstrap2 + JQuery
  • Logical layer: Struts2 + Spring Security
  • Persistence Layer: CRUD + Memcached + MongoDB

We can use it to quickly build easily maintainable and extensible business web applications.

About presentation layer

Bootstrap is a powerful front-end framework for faster and easier web development. It contains a 12-column responsive grid, dozens of components, JavaScript plugins, typography, form controls. Except for JQuery, underscore.js is used. It is a utility-belt library for JavaScript that provides a lot of the functional programming support on collections, arrays and javascript templating.

Generally, We use bootstrap to build static web page quickly and use jquery + underscore.js to add interface interaction. Then we will change the static pages to freemarker pages.

About logical layer

Logical layer is built on struts2, which is a MVC framework.

  • Models should always be defined in xml schema(.xsd), then generate them into java bean objects by castor.
  • Views should always be built with freemarker, which is a Java-based template engine that is a great alternative to JSP. Other result types are supported, such as json or XML.
    • Freemaker demo, please refer to DemoAction.returnPage().
    • Json demo, please refer to DemoAction.returnJson().
    • String demo, please refer to DemoAction.returnString().
    • XML demo, please refer to DemoAction.returnXml().
  • Controllers should always be implemented in struts2 actions. We can use the java bean objects generated from xsd to implement the business logics.

Spring Security is a powerful and highly customizable authentication and access-control framework.

About persistence layer

The java bean objects generated from xsd will be saved into database directly. MongoDB is used the store documents and indexes.

Because MongoDB used memory directly, If you application and db are in one server, no need to setup memcache.

Please refer to this demo for simple db access, Demo.
